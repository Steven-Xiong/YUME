## 1. Project Overview

**YUME 1.5** is an interactive world generation system built on top of the Wan2.2-5B pretrained model. It supports generating **infinitely autoregressive, explorable dynamic worlds** from a single image or text prompt, with intuitive camera control via keyboard (W/A/S/D) and mouse (↑↓←→).

### Three Generation Modes

| Mode | Description |
|------|-------------|
| Text-to-World (T2V) | Generate an explorable world from a text description |
| Image-to-World (I2V) | Generate a video world from a single static image |
| Event Editing | Trigger in-scene events via text (e.g., "A ghost appeared") |

### Key Performance Metrics

| Model | Time(s)↓ | Instr. Following↑ | Subject Consistency↑ | Background Consistency↑ |
|-------|----------|-------------------|----------------------|-------------------------|
| Wan-2.1 | 611 | 0.057 | 0.859 | 0.899 |
| MatrixGame | 971 | 0.271 | 0.911 | 0.932 |
| Yume 1.0 | 572 | 0.657 | 0.932 | 0.941 |
| **Yume 1.5** | **8** | **0.836** | **0.932** | **0.945** |

**Highlights**: Inference speed improved **70×**, instruction-following ability up 27% (0.657→0.836), achieving **12 FPS at 540P** on a single A100 GPU.

### 2.1 TSCM — Joint Temporal-Spatial-Channel Modeling (Section 4.2)

TSCM is the central architectural innovation of Yume1.5. It solves the fundamental problem of **computational cost growing linearly with the length of video history** during autoregressive generation. It consists of two complementary sub-mechanisms.

---

#### Sub-mechanism A: Temporal-Spatial Compression

**Core Idea**: The further a historical frame is from the current prediction, the more it is spatially compressed; the most recent frames retain full resolution.

Compression scheme defined by Paper Eq.3:

| History Frame Position | Patchify Rate (T, H, W) | Effective Token Reduction |
|------------------------|------------------------|--------------------------|
| t-1 to t-2 (most recent) | (1, 2, 2) | 4× tokens |
| t-3 to t-6 | (1, 4, 4) | 16× tokens |
| t-7 to t-23 | (1, 8, 8) | 64× tokens |
| Further history | (1, 16, 16) | 256× tokens |
| Initial frame | (1, 2, 2) | Specially preserved |

**Code Implementation** — `wan23/modules/model.py`, lines 486–494:

```python
# Multi-scale patch embeddings generated by interpolating pretrained weights
# No retraining required — reuse pretrained knowledge directly
self.patch_embedding_2x  = upsample_conv3d_weights_auto(self.patch_embedding, (1,4,4))
self.patch_embedding_4x  = upsample_conv3d_weights_auto(self.patch_embedding, (1,8,8))
self.patch_embedding_8x  = upsample_conv3d_weights_auto(self.patch_embedding, (1,16,16))
self.patch_embedding_16x = upsample_conv3d_weights_auto(self.patch_embedding, (1,32,32))
self.patch_embedding_2x_f = nn.Conv3d(
    self.patch_embedding.in_channels,
    self.patch_embedding.in_channels,
    kernel_size=(1,4,4), stride=(1,4,4),
)
```

Adaptive compression levels (`wan23/modules/model.py`, lines 599–718):

| Total History Frames | Compression Levels | Compression Ratio Distribution (old→new) |
|---------------------|--------------------|------------------------------------------|
| ≤ 6 frames | 3 levels | 1x, 2x, 1x |
| ≤ 22 frames | 4 levels | 1x, 4x, 2x, 1x |
| ≤ 86 frames | 5 levels | 1x, 8x, 4x, 2x, 1x |
| ≤ 342 frames | 6 levels | 2x, 16x, 8x, 4x, 2x, 1x |
| > 342 frames | 7 levels | 2x, 32x, 16x, 8x, 4x, 2x, 1x |

---

#### Sub-mechanism B: Channel Compression + Linear Attention

**This is the biggest architectural addition in Yume1.5 compared to Yume1.0.**

Historical frames are further compressed with an aggressive `(8, 4, 4)` Patchify rate and channel dimension reduced to 96, producing `z_linear`. These are then fused with prediction tokens via **Linear Attention** rather than standard Self-Attention.

Mathematical formulation (Paper Eq.4 & Eq.5):

$$o^l = \frac{\sum_{i=1}^N v_i^l \phi(k_i^l)^T \phi(q^l)}{\sum_{j=1}^N \phi(k_j^l)^T \phi(q^l)}, \quad \hat{o}^t = \text{Norm}(o^t) W^o$$

where $\phi$ is ReLU activation, reducing standard softmax attention complexity from $O(N^2)$ to $O(N)$.

**Design Rationale — Why Two Complementary Mechanisms?**

- Standard Attention is sensitive to the **number of tokens** (O(N²)) → reduce token count via **temporal-spatial compression**
- Linear Attention is sensitive to **channel dimension** → reduce channel dimension via **channel compression**
- Together they achieve joint temporal-spatial-channel compression while preserving quality

**Inference Speed Validation** (Paper Figure 7):

| Method | Inference Time as Video Length Grows |
|--------|--------------------------------------|
| TSCM (Yume1.5) | **Constant** (stable from 8+ video blocks onward) |
| Spatial Compression (Yume 1.0) | Gradually increases |
| Full Context Input | Sharply increases; truncated at 8 blocks |

**Ablation Results** (Paper Table 2):

| Compression Method | Instr. Following↑ | Subject Consistency↑ |
|--------------------|-------------------|----------------------|
| TSCM (Yume1.5) | **0.836** | 0.932 |
| Spatial Compression (Yume 1.0) | 0.767 | 0.935 |

### 2.2 Self-Forcing with TSCM — Mitigating Error Accumulation (Section 4.3)

**Problem**: During autoregressive generation, the model sees ground-truth frames at training time but its own generated frames (containing errors) at inference time. This **train-inference discrepancy** causes errors to accumulate over successive video chunks, degrading quality in later segments.

**Solution**: Inspired by Self-Forcing [Huang et al., 2025], but replacing KV-Cache with TSCM — enabling **unbounded historical context** without truncation, unlike the original Self-Forcing.

During training, the model's own predicted clean latent is used as the historical input for the next chunk:

**Training Code** — `fastvideo/distill_model.py`, lines 386–390:

```python
# Self-Forcing: use model's own predicted clean latent as historical context
if i+1 == 50:
    temp_x0 = latent[:,-9:,:,:] + (0 - sampling_sigmas[i]) * noise_pred_cond[:,-9:,:,:]
else:
    temp_x0 = latent[:,-9:,:,:] + (sampling_sigmas[i+1] - sampling_sigmas[i]) * noise_pred_cond[:,-9:,:,:]

latent = torch.cat([
    noise[:,:-9,:,:] * sampling_sigmas[max(i-1,0)] + (1 - sampling_sigmas[max(i-1,0)]) * model_input[:,:-9,:,:],
    temp_x0  # newly predicted clean latent ✅
], dim=1)
```

**Inference Code** — `webapp_single_gpu.py`, lines 836–861:

```python
# Update latent history (Self-Forcing core)
if is_i2v_mode or seg > 0:
    model_input_latent = torch.cat([
        model_input_latent[:,:-latent_frame_zero,:,:],  # previous history
        latent[:,-latent_frame_zero:,:,:]                # newly generated clean latent ✅
    ], dim=1)

# Update pixel-space history
if is_i2v_mode or seg > 0:
    model_input_de = torch.cat([
        model_input_de[:,:-frame_zero,:,:],
        video_tail_px[:,-frame_zero:,:,:]  # newly decoded video frames ✅
    ], dim=1)
```

**Validation** (Paper Figures 5 & 6, T2V mode, 6th video block):

| Metric | With Self-Forcing + TSCM | Without |
|--------|--------------------------|---------|
| Aesthetic Score | **0.523** | 0.442 |
| Image Quality | **0.601** | 0.542 |

### 2.3 Keyboard/Mouse Control via Natural Language Vocabulary (Section 3.1, Eq.1 & 2)

**Core Idea**: Map discrete keyboard (W/A/S/D) and mouse (↑↓←→) control signals to text descriptions via a **predefined vocabulary**, then append them to the generation caption, letting the language model learn to follow them.

**Caption Decomposition** (Yume1.5 architectural innovation):

- **Event Description**: Describes the scene/content; encoded by T5 only once at generation start
- **Action Description**: Keyboard + mouse controls; **finite set, precomputable and cacheable**, significantly reducing T5 overhead in subsequent inference steps

**Code Implementation** — `fastvideo/dataset/t2v_datasets.py`, lines 393–418:

### 2.4 GAN-based Distillation — 4-Step Inference Acceleration (Section 4.3)

**Goal**: Compress the 50-step diffusion model into a **4-step** generator while maintaining generation quality.

The paper adopts Distribution Matching Distillation (DMD), minimizing the KL divergence between the generated and real data distributions (Paper Eq.6). The code implements this via a more direct **GAN adversarial training** approach:

**Code Implementation** — `fastvideo/distill_model.py`, lines 320–354:

```python
if Distil:
    # 1) Compute clean latent (denoising result)
    model_denoing = xt - t * model_output        # x_0 = x_t - t*ε
    model_denoing = model_denoing[:,-9:]          # only judge the latest 9 frames

    # 2) Discriminator forward pass (Real vs Fake)
    with torch.autocast("cuda", dtype=torch.bfloat16):
        pred_real, pred_real_f, _ = discriminator(
            model_input_gan.permute(1,0,2,3).detach().reshape(b1*f1,c1,h1,w1), None)
        pred_fake, pred_fake_f, _ = discriminator(
            model_denoing.permute(1,0,2,3).detach().reshape(b1*f1,c1,h1,w1), None)

    # 3) Hinge Loss for discriminator
    loss_real = torch.mean(torch.relu(1.0 - pred_real)) + ...
    loss_fake = torch.mean(torch.relu(1.0 + pred_fake)) + ...
    loss_d = (loss_real + loss_fake) / 2.0
    loss_d.backward()

    # 4) Generator GAN loss (non-saturating)
    gan_loss = -torch.mean(pred_fake) - torch.mean(pred_fake_f)
    loss = loss + 0.01 * gan_loss  # weight 0.01
```

**Discriminator Architecture** — `ADD/models/discriminator.py`:

- **ProjectedDiscriminator**: Feature extractor based on DINO ViT-Small
- `SubPixelConvLayer`: Upsamples latent (16ch) to RGB space for discrimination
- Multi-scale discriminator heads: one head per DINO layer
- Combined Hinge Loss + Non-saturating Loss

**Key Hyperparameters**:

| Parameter | Value |
|-----------|-------|
| Discriminator learning rate | 1e-5 |
| GAN Loss weight | 0.01 |
| Sampling steps (training) | 50 steps |
| Sampling steps (after distillation) | **4 steps** |
| Frames judged per step | 9 frames (latest chunk only) |

### 2.5 InternVL3-2B — Multimodal Prompt Refinement (Section 3.1, Figure 2)

**Problem**: Short user prompts often fail to precisely describe the video events that should emerge from the given image. I2V generation requires descriptions grounded in the actual image content.

**Solution**: Use InternVL3 (the paper uses 78B; the code uses **2B** to save GPU memory) to automatically generate a detailed, image-grounded prompt from the input image + user prompt.

InternVL3-78B is also used to **re-annotate training data**, distinguishing:
- **T2V annotations**: Describe the scene and context (original captions retained)
- **I2V annotations**: Focus on dynamic events (regenerated by VLM, as shown in Figure 2)

**Code Implementation** — `webapp_single_gpu.py`, lines 258–547:

```python
# Memory-efficient strategy: reside on CPU, temporarily move to GPU when needed
caption_model = AutoModel.from_pretrained(
    "InternVL3-2B-Instruct",
    torch_dtype=torch.bfloat16,
    use_flash_attn=True,
).eval()
MODELS.caption_model = caption_model.cpu()  # keep on CPU to save VRAM

# At inference: image → dynamic tile preprocessing → temporarily on GPU
caption_model = MODELS.caption_model.to(MODELS.device, dtype=DTYPE)
question = (
    f'<image>\nWe want to generate a video using this prompt: "{user_prompt}". '
    'Please refine it for this image (<image>). Keep it one paragraph.'
)
out = caption_model.chat(MODELS.tokenizer, px, question, gen_cfg)
MODELS.caption_model.cpu()  # immediately return to CPU after use
```

**Image Preprocessing**: `dynamic_preprocess()` tiles the image into up to 12 patches of 448×448, compatible with InternVL's ViT visual encoder.

## 3. Training Stage Analysis

### Paper Description vs Code Implementation

The paper (Sections 4.3, 5.1.1) describes **2 formal training stages**. At the code level, these map to **3 separable sub-stages**:

---

### Stage 1: Foundation Model — `train-5b.py`

**Script**: `scripts/finetune/finetune-5b.sh`

```bash
torchrun --nproc_per_node 8 train-5b.py \
    --learning_rate=1e-5 \
    --max_train_steps=600000
    # No --MVDT, No --Distil
```

**What it does**:
- Trains on a mixed dataset: Real-world (Sekai-Real-HQ) + Synthetic (50K videos from Wan2.1 14B) + Event Dataset (4K curated I2V clips)
- **Alternating T2V/I2V training**: odd steps use T2V data, even steps use I2V data
- Standard Rectified Flow loss (no GAN, no masking)
- Paper trains for **10,000 iterations**

**Implementation Status**: ✅ Fully implemented  
> Note: MVDT-related code exists inside `train-5b.py` but is **commented out** (`# MVDT`), indicating it was tried in Stage 1 but later removed

---

### Stage 2: MVDT Self-Forcing — `distill_model.py --MVDT`

**Core**: Masked Video Diffusion Training (inspired by MDT from sail-sg/MDT)

At each training step, an additional **random token masking** forward + backward pass is performed:

```python
# fastvideo/distill_model.py
if MVDT:
    _, _, _, loss_dict_mask = denoiser.training_losses(
        transformer, model_input, arg_c,
        enable_mask=True   # triggers random masking + sideblock reconstruction
    )
    loss = loss_dict_mask["loss"].mean()
    loss.backward()  # separate backward pass
```

**Inside the model** — `wan23/modules/model.py`, lines 764–848:
- Randomly masks 30%~50% of tokens in the history portion
- A dedicated `sideblock` (independent WanAttentionBlock) reconstructs the masked tokens
- Reconstruction is inserted at the middle DiT block (after the `len_blocks//2`-th block)

**Why this matters**: Forces the model to generate coherent predictions even when historical context is **incomplete or corrupted** — directly aligned with Self-Forcing's needs, where historical frames contain generation errors equivalent to "partial corruption."

**Implementation Status**: ✅ Fully implemented

---

### Stage 3: GAN Distillation — `distill_model.py --MVDT --Distil`

**Script**: `scripts/finetune/finetune.sh` (**Stage 2 + Stage 3 combined in one run**)

```bash
torchrun --nproc_per_node 8 fastvideo/distill_model.py \
    --MVDT \    # Stage 2: Masked Self-Forcing
    --Distil \  # Stage 3: GAN distillation
    --learning_rate=1e-5 \
    --discriminator_learning_rate=1e-5
```

On top of Stage 2's Self-Forcing loss, adds GAN adversarial training with a projected discriminator, forcing **50-step → 4-step** inference capability. Paper trains for **600 iterations**.

**Implementation Status**: ✅ Fully implemented

```
Continuation mode (continue_from_last=True):
  model_input_de      ← LAST["last_model_input_de"]      pixel-space history
  model_input_latent  ← LAST["last_model_input_latent"]  latent-space history

I2V first generation:
  image → replicate 16 lead frames → pixel_values_vid (C, 16+33, H, W)
  VAE Encode → model_input_latent (C, Fz_hist, Hz, Wz)

T2V first generation:
  model_input_de / latent are empty; generation starts from pure noise

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 2: Build Prompt (Three-Layer Construction)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [1] Base prefix: "First-person perspective."

  [2] Action description (vocab mapping):
      camera_movement1="W" → "The camera pushes forward (W)."
      camera_movement2="→" → "The camera pans to the right (→)."

  [3] (Optional) InternVL3-2B refinement:
      image + user_prompt → InternVL → detailed one-paragraph description

  final_prompt = [prefix] + [action description] + [refined/user prompt]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 3: T5 Text Encoding (wan.generate)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  final_prompt → T5 Encoder (5B params, 4096-dim)
  ↓
  arg_c = {
    context:    text embeddings,
    seq_len:    token sequence length,
    grid_sizes: latent spatial dimensions,
    freqs:      RoPE positional encodings,
    ...
  }
  noise    = Gaussian noise (C, Fz_hist + latent_frame_zero, Hz, Wz)
  mask2    = temporal mask (0=history/fixed, 1=frames to generate)
  img_lat  = history latent conditioning

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 4: Chunk Generation Loop (sample_num iterations)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

for seg in range(sample_num):

  [A] Prepare latent input:
      seg==0 (I2V):  latent = [model_input_latent(history) | noise(new tail)]
      seg>0:         latent = [model_input_latent(accumulated) | new noise(tail)]

  [B] Diffusion sampling loop (steps=4):
      for i in range(4):

        ① Build timestep vector tvec:
           History frames: mask2=0 → t ≈ 0 (no noise, not updated)
           New tail frames: t = sampling_sigmas[i] * 1000

        ② Transformer forward pass (TSCM activates here):
           transformer(latent, t=tvec, **arg_c)
           │
           └─ TSCM: adaptive compression based on history length
              ≤ 6 frames  → patch_embedding_2x (2× compression)
              ≤ 22 frames → 4×/2× segmented compression
              ≤ 86 frames → 8×/4×/2× segmented compression
              ≤ 342 frames → 16×/8×/4×/2× segmented compression
              > 342 frames → 32×/16×/8×/4×/2× maximum compression
           │
           └─ Output: noise_pred (only the tail portion is used)

        ③ Euler step (only the tail frames are updated):
           new_tail = tail + (σ_{i+1} - σ_i) × pred_tail
           latent = concat([history frames | new_tail])
           # History frames remain frozen throughout

  [C] VAE decoding of new frames:
      video_tail = vae.decode(latent[:, -latent_frame_zero:])
      shape: (C, frame_zero, H, W) → pixel video frames

  [D] Self-Forcing history update (critical!):
      model_input_latent = concat([old history | latent's new tail])   ← clean latent ✅
      model_input_de     = concat([old history px | video_tail's tail]) ← decoded video ✅

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 5: Post-processing & Persistence
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  video_cat = concat(all video_tail chunks, dim=time)
  → write .mp4 file (16 FPS)

  LAST["last_model_input_latent"] = model_input_latent.detach()
  LAST["last_model_input_de"]     = model_input_de.detach()
  LAST["frame_total"]             = total frame count
  LAST["last_video_path"]         = video path
  LAST["last_prompt"]             = final_prompt
  ↑ Saved for the next "continuation" call
```

### Key Tensor Shape Tracing

```
# VAE compression: 8× spatial, 4× temporal
# Example: 33 frames at resolution 544×960
#   → latent_frame_zero = (33-1)//4 + 1 = 9
#   → latent shape: (C=16, 9, 68, 120)

# Chunk 1 (I2V):
#   model_input_latent: (16, 9_hist, 68, 120)    ← initial image latent
#   noise:              (16, 9+9_new, 68, 120)
#   latent (initial):   (16, 18, 68, 120)
#   → after 4-step denoising:
#   latent (clean):     (16, 18, 68, 120)
#   VAE decode tail:    (C, 33, 544, 960)         → pixel video

# Chunk 2 (Self-Forcing, history accumulated):
#   model_input_latent: (16, 9_hist+9_chunk1, 68, 120)  ← chunk1 appended
#   → TSCM selects 4-level compression (≤22 frames)

# Chunk 3+:
#   model_input_latent grows continuously; TSCM keeps inference time stable
```
